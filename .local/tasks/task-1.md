# Тренировочные программы — схема данных, API и фронтенд

## What & Why
Полностью переделать раздел «Спорт» — вместо простого списка тренировок создать систему тренировочных программ с ИИ-генерацией. Пользователь сможет создавать программы тренировок разных типов (зал, бег, плавание, йога и т.д.) с помощью ИИ, управлять ими, и назначать основную программу. Это фундамент для всех остальных подразделов (активная тренировка, история).

## Done looks like
- Новые таблицы в Supabase: программы тренировок (you_go_training_programs), упражнения программы (you_go_program_exercises), сессии тренировок (you_go_workout_sessions), результаты сетов (you_go_session_sets), тарифы пользователей (you_go_user_tiers)
- API для полного CRUD программ тренировок с привязкой к пользователю
- API для ИИ-генерации программы по заполненной форме (вид тренировки, тип, цели, уровень, длительность и т.д.)
- API для ИИ-модификации существующей программы (текстовые запросы на изменение)
- Контроль лимитов по тарифу: Free — 1 программа и 2 создания/неделю, Pro — 10 программ и безлимитные создания, Masters — 50 программ
- Фронтенд: полноценная страница «Программа тренировок» — основная программа отображается сверху со скроллом, список остальных программ (открыть/удалить/редактировать/сделать основной), кнопка создания
- Форма создания: красивый выбор вида тренировок (зал, бег, плавание, велосипед, йога и т.д.) с иконками, выбор типа (силовая, кардио, HIIT, функциональная, и т.д.), форма с обязательными и необязательными полями (уровень, цель, длительность, дни в неделю, ограничения, оборудование)
- После заполнения формы ИИ генерирует программу → пользователь может просмотреть, попросить ИИ текстом скорректировать, изменить вручную, утвердить и дать название
- Каждая программа содержит упражнения с: названием, описанием техники, советами по выполнению, количеством подходов/повторений/временем, целевыми мышцами, порядком
- У каждого вида/типа тренировок свой значок для удобной сортировки и визуального отличия
- Счётчик использованных/доступных программ, предупреждение при приближении к лимиту, предложение повысить тариф при достижении максимума
- Главный экран раздела «Спорт» переделан: статистика тренировок, кнопки перехода в подразделы, большая кнопка «Заниматься»
- Навигация с вкладками внутри раздела Спорт: Главная / Заниматься / Программы / История

## Out of scope
- Активная тренировка (пошаговое выполнение упражнений) — следующая задача
- История тренировок — следующая задача
- Аналитика, база упражнений, персональный ИИ-тренер — будущая работа
- Интеграция с фитнес-трекерами

## Tasks
1. **Новые таблицы в Supabase** — Создать таблицы you_go_training_programs (id, user_id, name, sport_type, training_type, is_primary, description, goal, level, days_per_week, duration_minutes, equipment, ai_generated, created_at), you_go_program_exercises (id, program_id, name, description, technique, tips, sets, reps, duration_seconds, rest_seconds, target_muscles, sort_order, exercise_type — 'sets_reps' или 'duration' или 'distance'), you_go_workout_sessions (id, user_id, program_id, started_at, completed_at, completion_percent, overall_rating, comment, ai_report), you_go_session_sets (id, session_id, exercise_id, set_number, weight, reps, duration_seconds, distance, rating_emoji, skipped), you_go_user_tiers (id, user_id, tier — 'free'/'pro'/'masters', programs_created_this_week, week_start). Для таблиц использовать тот же подход создания через /pg/query что и в существующих таблицах.

2. **Storage layer** — Расширить IStorage и SupabaseStorage: CRUD для программ (getPrograms, getProgram, createProgram, updateProgram, deleteProgram, setPrimaryProgram), CRUD для упражнений программы (getProgramExercises, addExercise, updateExercise, deleteExercise, reorderExercises), getUserTier, checkProgramLimits, incrementCreationCounter.

3. **API маршруты для программ** — REST API: GET/POST /api/training-programs, GET/PUT/DELETE /api/training-programs/:id, PATCH /api/training-programs/:id/set-primary, GET/POST/PUT/DELETE /api/training-programs/:id/exercises. Валидация через Zod, проверка лимитов тарифа при создании.

4. **API для ИИ-генерации программы** — POST /api/training-programs/generate: принимает параметры (вид, тип, цель, уровень, длительность, дни в неделю, оборудование, ограничения), формирует промпт, ИИ генерирует JSON с упражнениями (название, техника, советы, подходы, повторения, советы по весам, отдых), возвращает сгенерированную структуру (ещё не сохранённую). POST /api/training-programs/generate/modify: принимает текущую структуру + текстовый запрос на изменение, ИИ корректирует. POST /api/training-programs/generate/save: утверждение и сохранение.

5. **Фронтенд — навигация и структура** — Добавить вкладки внутри раздела Спорт (Главная, Заниматься, Программы, История). Переделать главный экран: статистика, кнопки в подразделы, большая кнопка «Заниматься». Создать отдельные компоненты для каждой вкладки.

6. **Фронтенд — страница «Программы»** — Основная программа вверху с упражнениями (скролл), кнопка «Начать тренировку». Ниже — список остальных программ с иконками типов, действиями (открыть/удалить/изменить/сделать основной). Счётчик программ (использовано/доступно), предупреждения по лимиту.

7. **Фронтенд — создание программы** — Красивый пошаговый мастер: шаг 1 — выбор вида тренировок с иконками (зал, бег, плавание, велосипед, йога, бокс, и т.д.), шаг 2 — выбор типа (силовая, кардио, HIIT, функциональная, растяжка), шаг 3 — форма с параметрами (обязательные выделены), шаг 4 — ИИ генерирует → предпросмотр → чат с ИИ для корректировки или ручное редактирование → утверждение и именование.

8. **Фронтенд — редактирование программы** — Открытие программы с возможностью ручного редактирования (перетаскивание упражнений, изменение параметров) и чат с ИИ для корректировки.
